🔑 КЛЮЧЕВЫЕ КОНЦЕПЦИИ:

✓ Архитектура: 3-уровневая (UI → API → DB)
✓ Мультитенантность: Полная изоляция данных через tenantId
✓ Soft Delete: Вместо физического удаления используется флаг
✓ Безопасность: RBAC, контроль доступа, аудит всех операций
✓ Масштабируемость: Оптимальное индексирование и пагинация

📋 ОСНОВНЫЕ СУЩНОСТИ:

• Organization - компании/организации
• Person - контакты (физ. лица)
• Pipeline - процессы продаж
• Stage - этапы конвейера
• Deal - сделки
• Lead - лиды (потенциальные контакты)
• Activity - активности (звонки, встречи, задачи)
• Email - электронные письма
• FieldDefinition - пользовательские поля
